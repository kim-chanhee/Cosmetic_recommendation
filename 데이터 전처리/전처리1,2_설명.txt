# 피부 질환 화장품 리뷰 데이터 전처리 과정 설명

## 전처리(1).ipynb - 기본 데이터 통합 및 정제

### 주요 단계:
1. **데이터 통합**
   - 크림 카테고리의 13개 CSV 파일을 하나로 통합
   - 각 파일에 product_name 컬럼 추가

2. **컬럼 정리**
   - 원하는 컬럼만 선택: product_name, reviewer_id, skin_type, skin_tone, skin_concern_1, skin_concern_2, rating, review_date, review 등
   - 컬럼명을 영어로 통일 (id → reviewer_id, tag → skin_type 등)

3. **데이터 정제**
   - 중복 리뷰 제거
   - 평점 형식 통일 (예: "5점만점에 5점" → "5")
   - summary_1, summary_2, summary_3, review 컬럼을 하나로 통합
   - 한글만 추출하여 정제

4. **결과**
   - 1,108개의 리뷰 데이터
   - 13개 컬럼으로 구성된 통합 데이터셋

## 전처리(2).ipynb - 토큰화 및 임베딩 모델 생성

### 주요 단계:
1. **토큰화 및 명사 추출**
   - 형태소 분석된 토큰에서 명사(NNG, NNP)만 추출
   - 새로운 컬럼 token_noun 생성

2. **N-gram 처리**
   - 3-gram: 3단어 조합을 하나의 의미있는 단어로 변환
   - 2-gram: 2단어 조합을 하나의 단어로 변환
   - 예: "수분크림", "속건조", "피부타입" 등

3. **불용어 제거**
   - 한국어 불용어 사전 활용
   - 브랜드명 제거 (에스트라, 올리브영, 피지오겔 등)
   - 1글자 단어 제거

4. **피부타입별 데이터 분리**
   - 복합성 피부 타입 데이터만 추출 (463개)

5. **임베딩 모델 생성**
   - Word2Vec과 FastText 모델 학습
   - 다양한 하이퍼파라미터 조합으로 실험
     * vector_size: [50, 100]
     * window: [3, 4, 5]
     * sg: [0(CBOW), 1(Skip-gram)]

6. **키워드 분석**
   - 복합성 피부 관련 키워드 분류:
     * 보습/수분: 보습, 보습력, 수분, 수분감
     * 유분기: 유분기, 기름
     * 피부 진정: 트러블, 진정, 여드름, 자극
     * 가성비: 가격, 용량
     * 주름: 주름, 탄력
     * 톤업: 미백, 기미, 톤업

### 최종 결과:
- 정제된 토큰화 데이터
- Word2Vec과 FastText 임베딩 모델
- 피부타입별 특화된 키워드 분석
- 감성분석을 위한 기반 데이터 구축

### 활용 목적:

- 고객 만족도 예측 모델 개발
